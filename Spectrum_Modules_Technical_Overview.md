# 🛠 Spectrum Trading: Обзор всех модулей и Техническая реализация

В данном документе представлен полный обзор экосистемы Spectr, включая описание функционала каждого модуля и архитектурные особенности их реализации.

---

## 1. Модуль AI-Сигналов (Signals Engine)

### Описание
Модуль для обнаружения высокодоходных торговых возможностей в реальном времени. Включает в себя "Degen Intel" (анализ ранних сигналов) и "TA Predictor" (технический анализ).

### Техническая реализация
*   **Scrapers (Ferocious Scout)**: Node.js скрипты, использующие `GramJS` или `Telegraf` для мониторинга закрытых Alpha-каналов в Telegram.
*   **On-chain мониторинг**: Интеграция с `DexScreener API` и `Helius` (для Solana) для отслеживания всплесков объема и ликвидности.
*   **Обработка данных**: `signalsController.js` агрегирует данные, присваивает им `Confidence Score` (на основе исторических данных и подтверждения объема) и сохраняет в БД через Prisma.
*   **Frontend**: Динамические карточки сигналов с эффектом "PricePulse" (изменение цвета при обновлении цены в реальном времени через WebSockets).

## 2. Модуль Новостей и NLP (News & Sentiment)

### Описание
Агрегатор крипто-новостей с автоматическим анализом настроений (Sentiment) и оценкой влияния на рынок (Impact).

### Техническая реализация
*   **Aggregation**: Парсинг RSS-фидов и API (CryptoPanic, CoinDesk, Twitter/X) с фильтрацией по 4 языкам (EN/RU/ES/PT).
*   **NLP Engine**: Интеграция с OpenAI API или локальным LLM для классификации новости:
    *   `Sentiment`: Bullish / Bearish / Neutral.
    *   `Impact`: Low / Medium / High.
*   **Linking**: Каждая новость привязывается к соответствующему тикеру (Asset), что позволяет модулю Сигналов учитывать новостной фон.
*   **Storage**: Таблица `News` в Prisma с уникальными ключами для предотвращения дублей.

## 3. Модуль Предсказаний (Spectrum Predictions)

### Описание
Платформа для ставок на исходы реальных событий в стиле Polymarket, полностью интегрированная в интерфейс Spectr.

### Техническая реализация
*   **Market Engine**: Система управления рынками (создание, лимиты ставок, расчет пулов).
*   **Trading Logic**: Покупка/продажа акций исхода ("Yes/No"). Использование CLOB (Central Limit Order Book) для сопоставления заявок пользователей.
*   **Oracles**: Система разрешения споров и закрытия рынков. Используются проверяемые источники данных (API крупных бирж, спортивные порталы) или Chainlink Oracles.
*   **UI/UX**: Использование `Framer Motion` для плавных переходов и `Lightweight Charts` для визуализации изменения вероятности исхода.

## 4. Модуль Арбитража (Arbitrage Monitoring)

### Описание
Поиск разницы в ценах на одни и те же активы между CEX (Binance, Bybit) и DEX (Raydium, Jupiter).

### Техническая реализация
*   **Real-time Feeds**: Подключение к WebSockets бирж для получения цен с минимальной задержкой.
*   **Calculation Engine**: Математическая модель, которая рассчитывает `Net Profit`, вычитая комиссии сетей (Gas), торговые комиссии бирж и потенциальное проскальзывание (Slippage).
*   **Alerting**: При нахождении связки с доходностью > 0.3% отправляется мгновенное Push-уведомление через Firebase.

## 5. Модуль Биллинга и доступа (Payments)

### Описание
Система монетизации с поддержкой крипто-платежей и подписочной моделью.

### Техническая реализация
*   **Gateway**: Интеграция с `NOWPayments` для приема USDT, SOL, ETH.
*   **Webhook Handler**: `paymentController.js` обрабатывает входящие вебхуки, проверяет подпись (HMAC) и обновляет статус пользователя в БД (`User.status = 'PRO'`).
*   **Access Control**: Middleware в Express проверяет уровень доступа пользователя перед отдачей данных из премиум-модулей.

## 6. Модуль Академии и Геймификации (Spectr Academy)

### Описание
Образовательная платформа для онбординга новых пользователей в мир трейдинга и предсказаний. Обучение построено на принципах "Learn-to-Earn" с выдачей наград за прохождение курсов.

### Техническая реализация
*   **Course System**: Структурированный контент (уроки, тесты) хранится в БД. Модели `Course` и `Lesson` в Prisma отслеживают прогресс каждого пользователя.
*   **Interactive Learning (Apply to Chart)**: Уникальная функция, позволяющая пользователю применить изученную концепцию (например, паттерн "Молот") прямо на живом графике в модуле Сигналов.
*   **Badge System**: После успешного прохождения тестов генерируется событие для n8n. Присваивается статус (`Badge`), который дает пользователю реальные преимущества (например, скидку 5% на Pro-план).
*   **Frontend**: Специальный раздел с прогресс-барами, визуализацией достигнутых бейджей и "залоченными" продвинутыми уроками.

---

## 7. Оркестрация и Автоматизация (n8n)

### Описание
Внешний слой логики для автоматизации сложных процессов: уведомления, геймификация, ребалансировка.

### Техническая реализация
*   **Workflow Trigger**: События в приложении (выполнение урока в Academy, новый сигнал) отправляют POST-запрос в n8n.
*   **Logic Steps**: n8n проверяет условия (например, "первый ли это бейдж пользователя?") и выполняет действия (выдача скидки, отправка поздравления в Telegram).
*   **Idempotency**: Каждое событие содержит `idempotencyKey` для предотвращения повторных срабатываний при сбоях сети.

## 🏗 Общий Технологический Стек

*   **Frontend**: React 18, Vite, Tailwind CSS (Vanilla CSS for Glassmorphism), Framer Motion.
*   **Backend**: Node.js, Express.
*   **Database**: Prisma ORM + SQLite (с возможностью миграции на PostgreSQL).
*   **Real-time**: Socket.io / WebSockets.
*   **PWA**: Service Workers для работы в режиме "Native App" на мобильных устройствах.
